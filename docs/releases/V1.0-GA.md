# Sarvaja v1.0-GA Release Notes

> **Release Date**: 2026-03-20
> **Version**: 1.0-GA (build 1.3.1)

## What is Sarvaja?

**Sarvaja** (Sanskrit: सर्वज, "All-Knowing") is a multi-agent governance platform that brings structure, traceability, and intelligence to AI-assisted software development. It combines TypeDB graph persistence, ChromaDB vector search, and Model Context Protocol (MCP) servers to create a unified governance layer for Claude Code workflows.

## Headline Feature: Sessions Management for Claude Code

Sarvaja v1's flagship capability is **full lifecycle management of Claude Code sessions** — automatically discovering, ingesting, and visualizing every development session as a first-class governance entity.

### What It Does

| Capability | Description |
|-----------|-------------|
| **Auto-Discovery** | Scans `~/.claude/projects/` on startup, finds all CC JSONL session files |
| **Streaming Ingestion** | Parses 600MB+ JSONL files with ~60MB peak memory via generator-based pipeline |
| **Transcript Viewing** | Color-coded conversation cards (user/assistant/tool/error) with 4-tier fallback |
| **Zoom-Level Detail** | Progressive disclosure: summary → tools → inputs → thinking content |
| **Entity Linking** | Connect sessions to rules, decisions, tasks, and evidence in TypeDB |
| **MCP Auto-Tracking** | Non-chat MCP tool calls automatically grouped into sessions |
| **Validation** | CVP heuristic checks detect anomalies (backfill detection, duration caps) |
| **Persistence** | TypeDB-first with disk-backed fallback — survives container restarts |

### Session Lifecycle

```
  Create          Record           Link            End             Analyze
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ POST     │───▸│ MCP tools│───▸│ Rules    │───▸│ PUT      │───▸│ Zoom     │
│ /sessions│    │ capture  │    │ Decisions│    │ /end     │    │ Transcript│
│          │    │ decisions│    │ Tasks    │    │ duration │    │ Validation│
│ Auto-ID  │    │ tasks    │    │ Evidence │    │ computed │    │ Evidence │
└─────────┘    │ tools    │    └──────────┘    └──────────┘    └──────────┘
               └──────────┘
```

### Transcript Fallback Chain

1. **JSONL** — Parse Claude Code session file directly
2. **In-Memory** — Synthetic transcript from `_sessions_store`
3. **Evidence** — Parse evidence `.md` file from `/evidence/`
4. **Empty** — Graceful empty state if nothing found

### Core Feature Areas

**Governance Rules Engine**
- 56 rules with semantic IDs (e.g., `SESSION-EVID-01-v1`, `TEST-GUARD-01-v1`)
- TypeDB persistence with inference rules
- CRUD via MCP tools and REST API
- Rule impact analysis with dependency graphs

**Agent Trust & Proposals**
- 5 governance agents with trust scoring
- Proposal creation, voting, and dispute resolution
- Trust leaderboard with activity tracking

**Task Management**
- Full backlog with phase/status/priority filters
- Task ↔ session ↔ rule ↔ evidence linking in TypeDB
- Gap tracking and auto-prioritization

**Audit Trail**
- All CRUD operations generate audit entries
- Correlation IDs, actor attribution, rule application tracking
- 7-day retention with entity/action/actor filters

**Data Integrity**
- 27 heuristic checks across TASK, SESSION, RULE, AGENT, CROSS-ENTITY, EXPLORATORY
- CVP (Continuous Validation Pipeline) with 3 tiers
- Automated sweep endpoint + real-time status

**Orchestrator Workflow**
- Continuous loop: GATE → BACKLOG → SPEC → IMPLEMENT → VALIDATE → CERTIFY → COMPLETE
- Dynamic ROI-based budget gating
- 3-tier Gherkin validation specs (Business → Technical → Low-level)

## Known Limitations

- Agent status is in-memory only (no TypeDB persistence for agent-status)
- Dashboard requires container restart after `agent/` code changes (no hot-reload)
- LangGraph replaced with MockStateGraph to prevent OOM on systems with <16GB RAM
- Session ID collisions possible when duplicate topics created on the same date without UUID

## Upgrade Path

Sarvaja v1 is the first GA release. No migration is required from development builds — the platform auto-discovers and ingests existing Claude Code sessions on startup.

```bash
# Quick start
git pull origin master
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
podman compose --profile cpu --profile dashboard-dev up -d
# Access dashboard at http://localhost:8081
```
