### REST API Request Models

**TaskCreate** (governance/models.py ~line 750):
- Present: `task_id, description, phase, status, priority, task_type, agent_id, body, linked_rules, linked_sessions, linked_documents, gap_id`
- **MISSING**: `workspace_id: Optional[str] = None`

**TaskUpdate** (governance/models.py ~line 765):
- **MISSING**: `workspace_id: Optional[str] = None`

**TaskResponse** (governance/models.py):
- `workspace_id` IS present (response shows the field, just always null via REST)

### Root Cause: MCP workspace_id Silent Failure

Code path: `mcp task_create` → `task_service.create_task(workspace_id=...)` → `TypeDB insert_task()` → creates task entity + attempts `workspace-has-task` relation insert → **FAILS** → entire transaction rolled back → no task persisted.

The MCP tool returns the in-memory representation (which shows success), but the TypeDB write failed, so `task_get` (which reads TypeDB) returns "not found".
