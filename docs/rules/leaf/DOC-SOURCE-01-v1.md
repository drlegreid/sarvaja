# DOC-SOURCE-01-v1: Official Documentation Sources First

**Category:** `documentation` | **Priority:** HIGH | **Status:** ACTIVE | **Type:** OPERATIONAL

> **Tags:** `documentation`, `research`, `official`, `sources`

---

## Directive

When researching technical topics, ALWAYS consult official documentation sources FIRST before third-party sources.

---

## Source Priority

| Priority | Source Type | Examples |
|----------|-------------|----------|
| 1 (Highest) | Official docs | docs.anthropic.com, code.claude.com, github.com/anthropics |
| 2 | Official GitHub | Issues, discussions on official repos |
| 3 | Official blogs | anthropic.com/engineering |
| 4 (Last) | Third-party | Stack Overflow, blogs, tutorials |

---

## Key Official Sources

| Topic | Official Source |
|-------|-----------------|
| Claude Code | https://code.claude.com/docs |
| Claude API | https://docs.anthropic.com |
| Claude Code Issues | https://github.com/anthropics/claude-code |
| MCP Protocol | https://github.com/modelcontextprotocol |
| Agent SDK | https://docs.claude.com/en/docs/agent-sdk |

---

## Anti-Patterns

| Don't | Do Instead |
|-------|-----------|
| Search random blogs first | Check official docs first |
| Trust outdated third-party info | Verify against official docs |
| Guess API options | Read official API reference |

---

## Validation

- [ ] Official source checked first
- [ ] Third-party info verified against official
- [ ] Links to official docs included in response
