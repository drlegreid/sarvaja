---

## Directive

Reports MUST be factual without flattery; prioritize accuracy over validation.

1. **Factual Content** - State facts, not opinions disguised as facts
2. **No Flattery** - Avoid excessive praise or validation-seeking language
3. **Accuracy First** - Prioritize correctness over user satisfaction
4. **Honest Assessment** - Report problems even if uncomfortable

---

## Validation

- [ ] Report contains only factual claims
- [ ] No excessive praise or flattery
- [ ] Problems clearly stated
- [ ] Recommendations based on evidence

---

## Anti-Patterns

| Don't | Do Instead |
|-------|------------|
| "Great work on..." | State what was accomplished |
| "As you wisely suggested..." | Reference the suggestion factually |
| Hide problems | Report issues clearly |
| Exaggerate success | Report accurate metrics |
