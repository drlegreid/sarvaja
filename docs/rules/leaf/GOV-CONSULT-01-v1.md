# GOV-CONSULT-01-v1: User Consultation on Priority Decisions

**Status:** ACTIVE | **Priority:** CRITICAL | **Category:** GOVERNANCE
**Created:** 2026-01-20 | **Author:** User Directive

---

## Directive

**When uncertain about priorities, scope, or deferral decisions, ALWAYS consult the user before proceeding.**

Agent and user are PARTNERS. Unilateral decisions on:
- Task deferral
- Priority assignment
- Scope reduction
- "Not worth the effort" judgments

...are PROHIBITED without explicit user consultation.

---

## Rationale

1. **Partnership Model**: User and agent are equal partners in decision-making
2. **Context Preservation**: User has business context agent lacks
3. **Accountability**: Shared decisions = shared ownership
4. **Trust Building**: Consultation demonstrates respect for user judgment

---

## Anti-Patterns (PROHIBITED)

| Anti-Pattern | Example | Correct Behavior |
|--------------|---------|------------------|
| Unilateral deferral | "This is over-engineered, deferring" | Ask: "Should we defer this or scope it differently?" |
| Dismissing work | "Not worth the effort" | Ask: "What priority should this have?" |
| Assuming user intent | "User probably doesn't need this" | Ask: "Is this feature important to you?" |
| Solo prioritization | "I'll do X before Y" | Ask: "Which should I tackle first?" |

---

## Consultation Template

When uncertain, ask:

```
I identified [SITUATION]. Options:
1. [Option A] - [tradeoffs]
2. [Option B] - [tradeoffs]
3. [Option C] - [tradeoffs]

Which approach do you prefer?
```

---

## Evidence Requirement

All deferral/priority decisions MUST include:
- User's explicit approval
- Rationale documented
- Task logged for future pickup (if deferred)
