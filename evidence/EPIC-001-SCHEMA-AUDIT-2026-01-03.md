**Status: INLINE TYPES (BY DESIGN)**

MCP tools use inline Python types in function signatures:

```python

    category: str,
    priority: str,
```

**Reason:** FastMCP decorator requires simple Python types, not Pydantic models.

**Conclusion:** MCP tools cannot use Pydantic models due to FastMCP limitations. This is architecturally correct.
